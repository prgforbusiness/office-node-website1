## メール送信設定手順

### 必要な情報

お問い合わせメールを受け取るために、以下の情報が必要です：

1. **送信先Gmailアドレス**：お問い合わせを受け取りたいメールアドレス

### 設定手順

#### Step 1: Resendアカウントの作成

1. [Resend](https://resend.com) にアクセス
2. 「Start for free」をクリックしてアカウント作成
3. メール認証を完了

#### Step 2: APIキーの取得

1. Resendダッシュボードにログイン
2. 左メニューから「API Keys」をクリック
3. 「Create API Key」をクリック
4. 名前を入力（例：「Office Node Website」）
5. 生成されたAPIキーをコピー（`re_` で始まる文字列）

#### Step 3: 環境変数の設定

プロジェクトルートに `.env.local` ファイルを作成し、以下を記入：

```
CONTACT_EMAIL=あなたのメールアドレス@gmail.com
RESEND_API_KEY=re_xxxxxxxxxxxxxxxxxxxxxxxxx
```

#### Step 4: 本番環境（Vercel）での設定

1. Vercelダッシュボードでプロジェクトを選択
2. 「Settings」→「Environment Variables」
3. 以下を追加：
   - `CONTACT_EMAIL`: お問い合わせを受け取るメールアドレス
   - `RESEND_API_KEY`: ResendのAPIキー

### 注意事項

- 無料プランでは月100通まで送信可能
- 本番環境では独自ドメインの設定を推奨（Resendで設定可能）
- `.env.local` ファイルは `.gitignore` に含まれているためGitにはコミットされません

## 画像・動画の配置場所

画像や動画は以下のフォルダに配置してください：

```
/public/
├── images/
│   ├── hero/
│   │   └── hero-main.jpg          # トップページのヒーロー画像（推奨: 800x600px以上）
│   ├── flow/
│   │   ├── step-01-hearing.jpg    # ヒアリング（推奨: 160x112px）
│   │   ├── step-02-design.jpg     # 設計
│   │   ├── step-03-implementation.jpg  # 実施
│   │   ├── step-04-retention.jpg  # 定着
│   │   └── step-05-growth.jpg     # 発展
│   ├── services/
│   │   ├── training.jpg           # 研修・セミナー
│   │   ├── consulting.jpg         # 顧問契約
│   │   └── dx-support.jpg         # DX導入支援
│   ├── about/
│   │   └── representative.jpg     # 代表者写真
│   └── why-us/
│       ├── reason-01.jpg          # 伴走型支援
│       ├── reason-02.jpg          # レベル対応
│       ├── reason-03.jpg          # 事前ヒアリング
│       ├── reason-04.jpg          # 実効性重視
│       └── reason-05.jpg          # 中小企業診断士の視点
└── videos/
    └── hero/
        └── hero-video.mp4         # ヒーロー動画（任意）
```

### 画像のガイドライン

- **フォーマット**: JPG, PNG, WebP（WebP推奨）
- **ヒーロー画像**: 1920x1080px 以上推奨
- **サムネイル**: 400x300px 程度
- **アイコン的な画像**: 160x112px 程度

### 画像を追加した後

画像を配置したら、各ページのコードでコメントアウトされている `<Image>` タグのコメントを解除し、プレースホルダーを削除してください。
